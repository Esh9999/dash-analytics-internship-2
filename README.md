# Dashboards + Analytics (Node.js + нативный JS)

Мини-проект для стажировки: 2 вкладки (**dashboards.html** и **analytics.html**), страница ошибки **error.html**, плюс серверная часть и обмен данными клиент ↔ сервер.

## Что сделано

- ✅ `package.json` (инициализация проекта)
- ✅ Навигация по вкладкам:
  - `/dashboards.html` — Дашборды
  - `/analytics.html` — Аналитика
- ✅ В случае ошибки (404) отдаётся `error.html` с возможностью вернуться к Дашбордам
- ✅ Клиент + сервер:
  - Сервер (Express) отдаёт страницы и API
  - Клиент (native JS) запрашивает данные у API и рисует график на `<canvas>`
- ✅ Файл `links.json` (туда вписываются ссылки на демо и код)

## Запуск локально

```bash
npm install
npm run dev
# затем открыть:
# http://localhost:3000/dashboards.html
```

## API

- `POST /api/event` — записать событие `{ page, action }`
- `GET /api/summary` — сводка: всего событий, по страницам, последние 5
- `GET /api/series?days=14` — временной ряд событий по дням

События хранятся в `data/events.json` (упрощённо, для учебной задачи).

## Как задеплоить (вариант: Render)

1) Залейте код в GitHub репозиторий.
2) В Render создайте **Web Service**:
   - Build command: `npm install`
   - Start command: `npm start`
3) После деплоя у вас будет публичный URL вида `https://...onrender.com`

> Если хотите именно GitHub Pages: там можно развернуть только статический фронтенд. Тогда сервер нужно размещать отдельно (Render/Railway/Fly.io) и включать CORS + прописать базовый URL API на клиенте.

## Где прописать ссылки

Откройте `links.json` и вставьте:

- `demoUrl` — ссылка на развернутое приложение
- `repoUrl` — ссылка на репозиторий с кодом
